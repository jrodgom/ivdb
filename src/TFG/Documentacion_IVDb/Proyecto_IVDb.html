INTRODUCCIÓN

Introducción al Dominio del Problema

En la actualidad, la industria de los videojuegos es uno de los sectores de entretenimiento con mayor crecimiento a nivel mundial. Con miles de títulos lanzados cada año en múltiples plataformas, los jugadores enfrentan el desafío de descubrir, organizar y gestionar información sobre los videojuegos que les interesan.

El proyecto IVDb nace como solución a esta problemática, ofreciendo una plataforma centralizada donde los usuarios pueden:

  • Consultar información detallada de videojuegos (descripción, género, fecha de lanzamiento, requisitos del sistema)
  • Mantener una lista personal de juegos favoritos
  • Escribir y leer reseñas de otros usuarios
  • Buscar y filtrar juegos según diversos criterios
  • Acceder a requisitos técnicos para PC

La plataforma se integra con la API de RAWG, una de las bases de datos de videojuegos más completas, permitiendo importar información actualizada de más de 800,000 títulos. Esto garantiza que los usuarios tengan acceso a un catálogo extenso y constantemente actualizado.

Descripción El sistema deberá respetar la siguiente regla de negocio: No pueden existir dos usuarios con el mismo nombre de usuario o dirección de correo electrónico. El sistema debe validar la unicidad durante el proceso de registro.

ARQUITECTURA

IVDb implementa una arquitectura de aplicación web moderna basada en el patrón Cliente-Servidor con API REST. Esta arquitectura separa claramente la lógica de presentación (frontend) de la lógica de negocio y acceso a datos (backend), permitiendo:

  • Desarrollo independiente y paralelo de frontend y backend
  • Escalabilidad horizontal de cada capa según necesidades
  • Posibilidad de crear múltiples clientes (web, móvil, desktop) consumiendo la misma API
  • Mantenimiento y actualización simplificados

Flujo de Comunicación

Ejemplo: Usuario añade un juego a favoritos

  1. Frontend: Usuario hace clic en botón "Añadir a Favoritos"
  2. Service (Frontend): favoriteService.addFavorite(gameId) realiza POST a la API
  3. Interceptor Axios: Añade automáticamente el JWT token en el header Authorization
  4. Backend API: Endpoint POST /api/favorites/ recibe la petición
  5. Autenticación: Django verifica el JWT token, identifica al usuario
  6. Permisos: Valida que el usuario esté autenticado
  7. Validación: Verifica que no exista ya el favorito (RN-001)
  8. ORM: Django crea registro en tabla favorite_favorite
  9. Respuesta: Backend devuelve JSON con el favorito creado (201 Created)
  10. Frontend: Actualiza UI mostrando el juego como favorito

Seguridad

Medidas Implementadas:

Aspecto Implementación
Autenticación JWT (JSON Web Tokens) con tokens de acceso (5min) y refresh (1 día). El token se almacena en localStorage del navegador.
Contraseñas Hashing mediante PBKDF2 con SHA256 (algoritmo por defecto de Django). Las contraseñas nunca se almacenan en texto plano.
CORS Configuración de django-cors-headers permitiendo solo orígenes específicos (localhost:5173 en desarrollo).
XSS Prevention Sanitización de HTML con DOMPurify en el frontend. Django escapa automáticamente el output en templates.
CSRF Protección CSRF deshabilitada para API REST (se utiliza JWT). Habilitada para Django Admin.
Validación Validación en frontend (React) y backend (Django Serializers) para evitar datos maliciosos.
Permisos Sistema de permisos de Django REST Framework: IsAuthenticated, IsAuthenticatedOrReadOnly, IsAdminUser.

Escalabilidad

La arquitectura actual permite escalar el sistema de las siguientes formas:

  • Migración a PostgreSQL: Cambio trivial gracias al ORM de Django
  • Separación de Servidores: Frontend y backend pueden desplegarse en servidores diferentes
  • Load Balancing: Múltiples instancias del backend detrás de un balanceador
  • CDN para Estáticos: Imágenes y assets del frontend pueden servirse desde CDN
  • Caché: Implementación de Redis para cachear queries frecuentes
  • Microservicios: Cada app Django podría convertirse en un microservicio independiente

Especificación de Actores del Sistema

ID Nombre Descripción
ACT-01 Usuario Anónimo Este actor representa a los visitantes de la plataforma que no han iniciado sesión. Pueden buscar y visualizar información de videojuegos pero no pueden interactuar con funcionalidades personales (favoritos, reseñas).

Flujo de Navegación Típico

Escenario 1: Usuario Anónimo busca un juego

  1. Usuario accede a la página principal (/)
  2. Ve el grid de juegos con paginación
  3. Introduce "Witcher" en la barra de búsqueda
  4. Sistema redirige a /search?q=Witcher
  5. Se muestran resultados filtrados
  6. Usuario hace clic en "The Witcher 3"
  7. Navega a /game/123
  8. Ve toda la información, reseñas, pero NO puede añadir a favoritos ni escribir reseña

Escenario 2: Usuario Registrado añade un juego a favoritos

  1. Usuario inicia sesión (/login)
  2. Es redirigido a la página principal
  3. Navbar ahora muestra "Favoritos" y "Perfil"
  4. Usuario navega a /game/456
  5. Ve botón de corazón vacío "Añadir a Favoritos"
  6. Hace clic en el corazón
  7. Corazón se rellena y aparece mensaje "Añadido a favoritos"
  8. Usuario puede ir a /favoritos para ver su lista

Escenario 3: Usuario escribe una reseña

  1. Usuario autenticado en página de detalle de juego
  2. Scrollea hasta sección de reseñas
  3. Ve formulario con selector de estrellas y textarea
  4. Selecciona 5 estrellas
  5. Escribe su opinión en el textarea
  6. Hace clic en "Publicar Reseña"
  7. Aparece spinner en botón
  8. Sistema valida y guarda la reseña
  9. Aparece mensaje verde "Reseña publicada exitosamente"
  10. La nueva reseña aparece en la lista

Escenario 4: Administrador añade un juego

  1. Admin inicia sesión con cuenta staff
  2. En navbar ve opción "Añadir Juego"
  3. Navega a /add-game
  4. Ve formulario con todos los campos del juego
  5. Completa: título, descripción, género, plataforma, desarrollador, fecha, imagen URL
  6. Hace clic en "Crear Juego"
  7. Sistema valida y crea el juego
  8. Redirige a /game/[nuevo-id]
  9. Ve el juego recién creado con botones "Editar" y "Eliminar" visibles

CONCLUSIONES

El desarrollo del proyecto IVDb - Base de Datos de Videojuegos ha permitido la implementación exitosa de una aplicación web completa que aborda la problemática inicial identificada: la falta de una plataforma centralizada para gestionar información de videojuegos, reseñas de usuarios y colecciones personales.

Objetivos Alcanzados

Se han cumplido satisfactoriamente todos los objetivos planteados al inicio del proyecto:

  • Sistema de autenticación completo: Implementación de registro, login y gestión de perfiles de usuario utilizando Django Rest Framework con autenticación JWT, garantizando seguridad y escalabilidad.
  • Catálogo de videojuegos: Desarrollo de un sistema de gestión de juegos con información detallada (descripción, requisitos del sistema, puntuaciones, géneros, plataformas) integrado con la API de RAWG para importación automatizada de datos.
  • Sistema de favoritos: Implementación de funcionalidad para que usuarios registrados puedan marcar juegos como favoritos y gestionar su colección personal.
  • Sistema de reseñas: Desarrollo de un sistema completo de valoraciones con puntuación de 1 a 5 estrellas y comentarios, permitiendo a la comunidad compartir opiniones sobre los videojuegos.
  • Panel de administración: Creación de funcionalidades exclusivas para usuarios administradores que permiten la gestión completa del catálogo (crear, editar, eliminar juegos).
  • Interfaz responsiva y moderna: Diseño de una interfaz de usuario intuitiva utilizando React 18 y TailwindCSS, adaptable a dispositivos móviles, tablets y escritorio.

Aprendizajes Técnicos

Durante el desarrollo del proyecto se han adquirido y consolidado conocimientos técnicos significativos:

  • Arquitectura Backend con Django: Profundización en el framework Django 5.2, incluyendo Django Rest Framework para APIs RESTful, sistema de autenticación con JWT, ORM para gestión de base de datos, y organización modular del código mediante apps Django.
  • Desarrollo Frontend con React: Uso avanzado de React 18 con hooks (useState, useEffect, useContext), React Router DOM para navegación SPA, gestión de estado con Context API, y manejo de peticiones asíncronas con Axios.
  • Integración de APIs externas: Implementación de scripts de importación desde RAWG API, manejo de respuestas JSON complejas, y procesamiento de datos para almacenamiento en base de datos.
  • Diseño responsivo: Aplicación de TailwindCSS para crear una interfaz moderna y adaptable, utilizando sistema de grid responsivo y componentes reutilizables.
  • Seguridad en aplicaciones web: Implementación de buenas prácticas de seguridad como autenticación JWT, encriptación de contraseñas con PBKDF2_SHA256, sanitización de HTML con DOMPurify, y configuración de CORS.
  • Trabajo con Git y GitHub: Gestión de versiones del proyecto, uso de ramas para desarrollo de funcionalidades, y documentación mediante README.md.

Desafíos Superados

A lo largo del desarrollo se presentaron diversos retos técnicos que fueron resueltos satisfactoriamente:

  • Integración Frontend-Backend: Configuración correcta de CORS, gestión de tokens JWT en localStorage, y manejo de estados de autenticación compartidos entre componentes mediante Context API.
  • Importación masiva de datos: Desarrollo de scripts de importación robustos capaces de manejar grandes volúmenes de datos desde RAWG API, con gestión de errores y rate limiting.
  • Renderizado de HTML dinámico: Implementación segura de renderizado de descripciones HTML provenientes de RAWG, utilizando DOMPurify para prevenir ataques XSS.
  • Gestión de requisitos del sistema: Diseño de estructura JSONField en Django para almacenar requisitos mínimos y recomendados de forma flexible, y componente React para renderizarlos en formato tabla comparativa.
  • Diseño responsivo complejo: Adaptación de componentes como GameFilters, SystemRequirements y navegación para funcionar correctamente en dispositivos móviles sin perder funcionalidad.

Trabajo Futuro y Mejoras Potenciales

Aunque el proyecto cumple con los objetivos establecidos, se identifican áreas de mejora y funcionalidades adicionales que podrían implementarse en futuras versiones:

  • Sistema de notificaciones: Implementar notificaciones push cuando se añaden nuevos juegos o cuando otros usuarios comentan en reseñas.
  • Sistema de recomendaciones: Desarrollar un algoritmo de recomendación basado en los juegos favoritos y reseñas del usuario, utilizando técnicas de machine learning.
  • Perfiles de usuario avanzados: Permitir que los usuarios suban screenshots, creen listas personalizadas de juegos, y sigan a otros usuarios.
  • Integración con más APIs: Conectar con otras fuentes de datos como Steam API, Twitch API para mostrar streams en vivo, o YouTube API para trailers oficiales.
  • Sistema de logros/achievements: Gamificación de la plataforma con logros por número de reseñas, juegos marcados como favoritos, etc.
  • Búsqueda avanzada: Implementar filtros adicionales como rango de fechas, puntuación mínima, desarrollador específico, y ordenamiento por múltiples criterios.
  • Migración a base de datos PostgreSQL: Para entornos de producción con mayor carga, migrar desde SQLite a PostgreSQL para mejor rendimiento.
  • Testing automatizado: Ampliar la cobertura de tests unitarios e integración tanto en backend (pytest) como en frontend (Jest/React Testing Library).
  • Deployment profesional: Despliegue en servicios cloud como AWS, Heroku o Vercel con configuración de CI/CD.
  • Modo oscuro: Implementación de un switch para alternar entre tema claro y oscuro, almacenando preferencia del usuario.
